## weixue-dog-nano-code 代码库总结

该 `weixue-dog-nano-code` 目录包含一个结构良好的、基于 Python 的 12 舵机四足机器人控制系统。其目的是在上位机（如 Jetson Nano 或 PC）上运行，并作为机器人的“大脑”，执行高级计算并将低级命令发送到微控制器（很可能是 `weixue-wheeled-dog` 项目中的 STM32）。

### 架构
架构是模块化的：
1.  **配置 (`config.json`, `robot_config.py`):** 定义了机器人的物理尺寸、舵机限制和电机方向。
2.  **运动学模型 (`kinematics_v5.py`):** 提供了核心逆运动学（IK）以从期望的足端位置计算关节角度，以及用于仿真的正运动学（FK）。
3  **通信与控制:** 使用自定义的串口协议（115200 波特率，带包头和校验和的报文）与 STM32 进行通信。主要的舵机控制命令将多个舵机的 ID、位置、速度和加速度捆绑到一个数据包中。
4.  **应用程序:**
    *   `servo_control_center.py`: 一个 `tkinter` GUI，用于手动控制、调试以及实时可视化舵机反馈（位置、负载）和 IMU 数据。
    *   `simulation_ik_v5.py`: 一个基于 `matplotlib` 和 `tkinter` 的仿真器，用于测试和可视化由 IK 引擎计算的静态姿态。
    *   `test_*.py` 脚本（例如 `test_trot_in_place.py`）：独立的脚本，通过生成随时间变化的轨迹并以高频（50Hz）控制循环使用运动学模块来实现动态步态（如小跑）。

### 与 STM32 的交互
上位机代码发送诸如 `CMD_TYPE_SERVO_CTRL`（包含所有 12 个舵机的目标位置）和 `CMD_TYPE_TORQUE`（用于使能/禁用电源）等命令。它期望 STM32 解析这些数据包，相应地控制舵机，并发送回包含实时舵机数据 (`FB_TYPE_SERVO_INFO`) 和 IMU 读数 (`FB_TYPE_SENSOR_IMU`) 的反馈数据包。上位机代码从不直接与硬件交互；它将所有实时任务卸载到 STM32。

### 主要功能
-   **低级控制:** 设置单个舵机的位置，并带有速度和加速度限制。
-   **姿态控制:** 计算并命令整个机器人采用特定的静态姿态（例如，站立、躺下）。
-   **动态步态:** 使用基于相位的中央模式发生器生成连续的行走/小跑运动。
-   **仿真与调试:** 提供强大的 GUI 工具，用于离线仿真和在线调试，这大大加快了开发速度。
